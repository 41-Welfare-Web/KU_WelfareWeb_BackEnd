## 3. 물품 이미지 업로드 및 DB 복구

`restore_items.ts`가 `itemImages` 테이블을 복구하지 않아 DB에 갤러리 이미지가 전혀 없던 상태를 발견하고 전체 복구.

**발견된 문제:**
- `itemImages` 테이블: 53개 물품 전부 0개 (완전히 비어있음)
- `items.imageUrl` null인 물품: 천막(ID:1), 포토월(ID:47)

**이미지 파일 명명 규칙 확립:**
```
{itemCode}_{순서}.{확장자}
예) 101_1.jpg, 206_2.jpg, 412_1.png
```

**신규 이미지 추가:**
- 천막: `101_1.jpg` (기존 없음 → 신규 촬영 후 추가)
- 포토월: `412_1.png` (기존 없음 → 신규 촬영 후 추가)

**이미지 폴더:** `Project/data/2026_03_23_Item_Image/`

**`prisma/upload_images.ts` 스크립트 작성 (재사용 가능):**
1. 로컬 이미지 → Supabase `rental-web` 버킷 `items/` 경로에 upsert 업로드
2. 파일명 파싱 → itemCode로 itemId 조회 → `itemImages` 테이블 upsert
3. order=1 이미지를 `items.imageUrl` 대표 이미지로 자동 업데이트

**실행 결과:**
