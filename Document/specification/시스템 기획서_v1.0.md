### **RentalWeb 서비스 기획서 (v1.0)**

**1. 프로젝트 개요 및 목표**

'RentalWeb'은 특정 단체(예: 대학교) 내 구성원들을 위한 물품 대여 및 플로터 출력 주문 서비스입니다. 기존의 오프라인으로 처리되던 대여/주문 절차를 온라인으로 전환하여 사용자의 편의성을 높이고, 관리자의 업무 효율을 극대화하는 것을 목표로 합니다.

*   **핵심 목표:**
    *   **사용자:** 언제 어디서든 실시간으로 물품 재고를 확인하고, 간편하게 대여 및 주문을 신청할 수 있는 환경 제공.
    *   **관리자:** 분산된 대여 정보를 중앙에서 통합 관리하고, 주문 승인 및 물품 관리 업무를 자동화하여 업무 부담 경감.
    *   **시스템:** 투명한 대여 이력 추적과 정확한 재고 관리를 통해 자원의 유실 및 오남용 방지.

**2. 주요 기능 정의**

| 구분 | 기능 | 상세 설명 |
| :--- | :--- | :--- |
| **공통** | **계정 관리** | 회원가입, 로그인, 아이디/비밀번호 찾기, 역할 기반 접근 제어 |
| **사용자** | **물품 대여** | - **탐색:** 키워드/카테고리 검색, 인기순/이름순 정렬<br>- **확인:** 상세 정보 및 날짜별 실시간 재고 캘린더 확인<br>- **예약:** 장바구니 기반 다중 물품 동시 예약, 기간/수량 변경, 예약 취소 |
| | **플로터 주문** | - **신청:** 파일 업로드 및 유효성(PDF, 페이지 수) 자동 검증<br>- **안내:** 근무일 기준 수령 예정일 자동 계산 및 안내<br>- **결제:** 유/무료 조건 자동 판별 및 유료 시 입금 내역 업로드 |
| | **마이페이지** | 내 정보 수정, 회원 탈퇴, 나의 대여/주문 현황 조회 |
| **관리자** | **대시보드** | 주요 현황(오늘의 대여/반납 건, 신규 주문 등) 요약 |
| | **물품 관리** | 물품 등록(CRUD), 재고 수량 관리 |
| | **대여 관리** | 사용자 대여 건 상태 변경(대여 중, 반납, 취소), 수동 예약 생성, 불량/취소 사유 기록 |
| | **플로터 주문 관리** | 주문 건 상태 변경(승인, 반려, 인쇄 완료), 반려 사유 기록 |
| | **사용자 관리** | 전체 사용자 목록 조회, 역할(Admin/User) 변경 |
| | **운영 관리** | 휴무일 및 운영 일정 캘린더 설정 |

**3. 화면 흐름 설계 (Information Architecture)**

사용자가 서비스를 이용하는 과정에서 마주할 주요 화면들의 구조입니다.

```
1. 시작
   ├─ 1-1. 메인 (로그인 전)
   └─ 1-2. 회원가입

2. 사용자 (로그인 후)
   ├─ 2-1. 물품 목록 (메인 화면)
   │   └─ (검색, 정렬, 카테고리 필터)
   ├─ 2-2. 물품 상세
   │   └─ (재고 캘린더, 장바구니 담기, 미니 장바구니)
   ├─ 2-3. 장바구니
   │   └─ (대여 기간 선택 캘린더, 품목별 수량/삭제)
   ├─ 2-4. 대여 확정
   │   └─ (최종 정보 확인 및 예약)
   ├─ 2-5. 플로터 주문
   │   └─ (파일 업로드, 정보 입력)
   └─ 2-6. 마이페이지
       ├─ 2-6-1. 나의 대여/주문 현황
       └─ 2-6-2. 회원정보 수정

3. 관리자 (로그인 후)
   ├─ 3-1. 관리자 대시보드
   ├─ 3-2. 물품 관리
   │   ├─ 3-2-1. 물품 목록 조회
   │   └─ 3-2-2. 물품 등록/수정
   ├─ 3-3. 대여 관리
   │   └─ (전체 대여/예약 목록, 상태 변경)
   ├─ 3-4. 플로터 주문 관리
   │   └─ (전체 주문 목록, 상태 변경)
   ├─ 3-5. 사용자 관리
   └─ 3-6. 운영일 관리 (캘린더)
```

**4. 주요 정책**

*   **대여 정책:**
    *   모든 대여는 장바구니를 통해 이루어지며, 각 품목별로 대여 기간을 설정해야 함.
    *   예약은 오늘로부터 최대 2개월 이내의 날짜만 가능.
    *   주말, 공휴일 및 관리자가 지정한 휴무일에는 대여/반납 불가.
    *   '예약' 상태에서만 사용자가 직접 수정/취소 가능.
*   **플로터 주문 정책:**
    *   신청 시 PDF 파일만 업로드 가능하며, 시스템이 페이지 수를 자동 검증함.
    *   수령일은 신청일 기준 '근무일 2일 뒤'로 자동 지정됨.
    *   '주문 대기' 상태에서만 사용자가 직접 취소 가능. 수정은 불가.
*   **알림 정책 (SMS):**
    *   **물품:** 예약 완료, 반납일 D-1, 연체 시 알림.
    *   **플로터:** 주문 반려(사유 포함), 인쇄 완료(수령 안내) 시 알림.

**5. 기술 스택 제안**

*   **프론트엔드:** React / 배포: Vercel
*   **백엔드:** NestJS (TypeScript), Prisma (ORM) / 배포: Railway
*   **데이터베이스:** Supabase (PostgreSQL 호스팅)
*   **인증:** JWT (NestJS + Passport)
